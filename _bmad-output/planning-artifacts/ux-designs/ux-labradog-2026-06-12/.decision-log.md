# Decision Log — UX Labradog (2026-06-12)

## Intake (brain dump de Nelson, 2026-06-12)

- **Detonante**: el diseño actual es "básico/horrible" — shadcn por defecto sin identidad; faltan botones "volver" en varios flujos; sin patrón de navegación consistente.
- **Dirección visual de Nelson (canónica)**: verdes pasteles o azules que reflejen CUIDADO; imágenes/iconografía de mascotas; aspecto moderno y suave.
- **Alcance**: app shell + navegación consistente (volver SIEMPRE, navegación móvil del paseador, admin desktop) + identidad visual aplicada a las pantallas ya construidas: portada, login, admin (home/equipo/tutores/perros/paseadores), paseador (mi día/mi capacitación/etapa).
- **Mandato de proceso**: en cada análisis futuro se DEBE evaluar UX y cuidar lo visual (guardarraíl pendiente de configurar fuera de este spec).

## Contexto del proyecto (fuentes)

- project-context.md: UI 100% es-CL; paseador móvil-primero en la calle (zonas táctiles ≥48px, un toque, teclado solo si se invoca); admin desktop; shadcn/ui + Tailwind 4 (Foundation hereda de shadcn — los tokens DESIGN.md extienden sus variables CSS).
